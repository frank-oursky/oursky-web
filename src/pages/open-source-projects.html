Ourskyers are a socially progressive team of geeks and craftsmen whose big hearts are committed to make the world a better place using technology. That's why we make our projects open source as much as we can. We have sponsored numerous technology and development community events, including COSCUP, Hong Kong Open Source Conference, PyCon APAC, and SITCON.

Why open-source? Having other versatile developers across different fields, countries, and backgrounds contributing to your idea makes a huge difference. You get to breed more ideas, work on exciting projects you may not have thought of doing, collaborate with experts, and earn a badge of honor among the community. The list goes on

Want to contribute to Oursky’s open-source projects?

Check out Github repos!

For Oursky, our open-source projects is our manifesto: harnessing technology and the digital space to educate, communicate, collaborate, and transform societies for the better.

While we regularly contribute to other open-source projects, we’ve also released our own projects over the years. If you’re interested in contributing, check them out below. You can also visit our Code Blog to learn more about them.

Highlight of Our Open Sources Project


  • Authgear - OIDC compatible auth services for Web / Mobile apps

  • devsecops-ci — DevSecOps implementation at Oursky, providing an easy way to invoke a checker toolset to enhance security automatically

  • react-messageformat — Provides better React app internationalization

  • stylelint-oursky — Helps ensure that #JavaScript projects follow a uniform and consistent lint style

Kubernetes / Continuous Integration (CI)


  • kubernetes-github-authn: GitHub token authentication for Kubernetes

  • devsecops-ci — DevSecOps implementation at Oursky, providing an easy way to invoke a checker toolset to enhance security automatically

  • flutter-docker — Dockerfile for building a Docker image for CI use

  • devsecops-secret — Secret generators for CI and dev env

  • DevSecOps-ignore — Can apply to all repositories within the organization; if there is an ignore file for each repository, that file should contain basic ignore entries

  • travis-ios-script — Reusable iOS Script for Travis CI integration with TestFlight, Crittercism, and HockeyApp support

  • gogocyclo — Tool intended to be used in CI to ignore certain results from gocyclo

  • slack-dsym-bot — Slackbot for iOS crash report via Sentry

Web / Mobile


  • react-native-oursky — React Native Scaffolding commonly used by Ourskyers

  • react-messageformat — Provides better React app internationalization

  • Redux - Swift implementation of Redux

Other Tools


  • stylelint-oursky — Helps ensure that #JavaScript projects follow a uniform and consistent lint style

  • gh-html — Github Pages alike for private repository

  • aws-site-manager - Very simple command-line interface (CLI) to create S3/Cloudfront static website

  • dotfiles - Various dotfiles that can be used company-wide

  • parse-server-sendgrid-adapter - Simple sendgrid adapter for parse server

  • norecaptcha - Python client for Google’s No CAPTCHA reCAPTCHA services
